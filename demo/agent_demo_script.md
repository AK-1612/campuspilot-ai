## Demo Flow

### Scenario 1 — Wheelchair User (30 seconds)

**Setup:** A wheelchair user scans QR code `#102` at the **Main Entrance** and asks to navigate to the library.

**Step 1 — QR Scan:**  
The user opens CampusPilot, scans QR #102. The app calls `/qr?id=102`, which resolves to `Main Entrance`.

**Step 2 — User Query:**  
User types: *"Take me to the library."*

**Step 3 — Agent Reasoning (narrate this):**
> "The agent receives the query. It first classifies the intent as `navigate`. It then sees the active profile is `Wheelchair`, which sets `avoid_stairs=True` and `prefer_elevators=True`. It calls `qr_lookup('102')` to confirm the start node is `Main Entrance`, then calls `route_query(start='Main Entrance', end='Library', profile='Wheelchair')`. Neo4j returns a path that avoids all staircase edges and uses Elevator B."

**Step 4 — Response:**  
> *"You are at the Main Entrance. Go straight to Elevator B (10 metres ahead on your right). Take it to Floor 2. Turn left — the library entrance will be 15 metres ahead."*

### Scenario 2 — Vision-Impaired User (30 seconds)

**Setup:** A vision-impaired professor needs to find the nearest accessible restroom.

**Step 1 — User Query:**  
User types: *"Where is the nearest restroom?"*

**Step 2 — Agent Reasoning (narrate this):**
> "Intent is classified as `find_facility`. The active profile is `Vision-impaired`, setting `detailed_landmarks=True`. The agent calls `route_query` with the enriched constraint. Neo4j returns a path annotated with tactile-path waypoints and audible landmarks instead of visual markers."

**Step 3 — Response:**  
> *"The nearest restroom is 30 metres ahead. Walk straight until you feel the textured floor strip, then turn right at the water fountain (you will hear it). The restroom door is 5 metres on your left."*

---

### Scenario 3 — Emergency Override (30 seconds)

**Setup:** Any user types a distress message.

**Step 1 — User Query:**  
User types: *"I fell down the stairs, I need help!"*

**Step 2 — Agent Reasoning (narrate this):**
> "The emergency intent is classified immediately, overriding all profile routing logic. The agent calls `flag_obstacle()` to mark the staircase on the shadow map. It simultaneously returns the emergency response — no Neo4j route lookup is performed."

**Step 3 — Response:**  
> *"Emergency detected. Campus security has been notified. Stay where you are. Help is on the way. Your last known location: Main Entrance (QR #102)."*

## Key Points to Emphasise to Judges

1. **Tool selection is autonomous** — the LLM decides which tool to call, in what order, without hard-coded if/else logic.
2. **Profile constraints are injected, not prompted** — the routing constraints from `profile_handler.py` are passed directly to the Neo4j query, not described in the system prompt.
3. **Emergency is pre-LLM** — the heuristic classifier catches emergencies before the LLM even runs, ensuring sub-100ms response to SOS queries.
4. **Memory is stateful** — the agent remembers your last QR scan, so if the user asks a follow-up question, it doesn't need to rescan.

## Q&A Prep — Likely Judge Questions

| Question | Answer |
|---|---|
| Walk me through what happens when a wheelchair user types 'take me to the library'. | Intent=navigate → profile=Wheelchair → constraints={avoid_stairs, prefer_elevators} → qr_lookup → route_query → Neo4j path → formatted response |
| Why Groq with Llama 3.3 70B and not Claude / GPT-4o? | Groq delivers Llama 3.3 70B at over 250 tokens per second, ensuring sub-second real-time navigation reasoning. Llama 3.3 70B also matches frontier models in reasoning quality while remaining extremely cost-effective. |
| What if the LLM hallucinates a room number? | The room ID is resolved from Neo4j, not generated by the LLM. The LLM only formats the response; the path data comes from the graph. |
| How does the agent decide which tool to call first? | LangChain's ReAct loop — the LLM reasons about what information it needs next, then acts. It will always call `qr_lookup` before `route_query` if it doesn't have a start node. |
| What is the ReAct pattern? | Reason + Act. The LLM alternates between thinking about what to do next and executing a tool action, observing the result, and repeating until it has enough information to respond. |
| How does memory work across turns? | `AgentMemory` stores the conversation history, current profile, and last QR location in-memory per session. Each new message appends to the history before being sent to the LLM. |
